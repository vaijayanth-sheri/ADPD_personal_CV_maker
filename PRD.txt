# Product Requirements Document (PRD)

## Project Name

Application Document Personalization Dashboard (ADPD)

Version: V1

Status: Design Phase

Owner: Vaijayanth Sheri

---

# 1. Product Vision

The Application Document Personalization Dashboard is a personal productivity platform designed to eliminate manual editing of CVs and cover letters.

The platform allows users to store professionally formatted DOCX templates and automatically apply structured content updates provided through Markdown files.

The system focuses exclusively on document personalization and formatting preservation.

No AI generation is performed within the platform.

The objective is to reduce repetitive editing work from 20–30 minutes per application to less than 60 seconds.

---

# 2. Problem Statement

Current workflow:

1. User receives customized CV content.
2. User receives customized Cover Letter content.
3. User manually opens CV template.
4. User manually copies content.
5. User manually updates sections.
6. User checks formatting.
7. User exports final files.

Pain Points:

* Repetitive copy-paste work
* Formatting mistakes
* Missed section updates
* Inconsistent document versions
* Time-consuming editing process

Desired Outcome:

User uploads a structured Markdown change file and receives:

* Updated CV DOCX
* Updated Cover Letter DOCX

while preserving the original template formatting exactly.

# 4. Core Principle

The dashboard must preserve:

* Fonts
* Font sizes
* Colors
* Margins
* Icons
* Tables
* Header/Footer
* Page breaks
* Images
* Layout structure
* Section ordering

Only text content inside predefined placeholders may change.

No formatting modifications are allowed.

# 7. Markdown Structure

The dashboard accepts only structured markdown.

Example:

# CV

## SUMMARY

Updated summary text

## PROJECT_PVLIB

Updated project description

## SKILLS

Python
Dash
Streamlit

# COVER_LETTER

## INTRODUCTION

Custom introduction

## MOTIVATION

Custom motivation section

## CLOSING

Closing paragraph

---

System must reject malformed structures.

Validation feedback should be human-readable.

---

# 8. Placeholder System

Template placeholders are mandatory.

Examples:

{{SUMMARY}}

{{SKILLS}}

{{PROJECT_PVLIB}}

{{PROJECT_ENERGYSIM}}

{{INTRODUCTION}}

{{MOTIVATION}}

{{CLOSING}}

The engine replaces only matching placeholders.

No free-text searching.

No AI matching.

No guessing.

---

# 9. Dynamic Cover Letter Sections

MOTIVATION

PROJECT_MATCH

CLOSING

System must support hiding unused sections without breaking formatting.
